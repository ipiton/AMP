## Goal

Translate the spec into an execution checklist.

## Do

1. Require `Spec.md`.
2. Create `tasks/<TASK-ID>/tasks.md`.
3. Split work into vertical slices when needed.
4. Include implementation, tests, docs, and finalization steps.
5. Make blockers and open assumptions explicit.
