# /start-task

Use:

- `skills/solo-kanban-core/SKILL.md`
- `skills/solo-kanban-planning/SKILL.md`

## Goal

Start one queued Solo Kanban task.

## Do

1. Read `WORKFLOW.md`, `docs/06-planning/NEXT.md`, and current git branch/state.
2. Identify the task from the user request or the top queued item.
3. Move the task from Queue to WIP in `NEXT.md`.
4. Create `tasks/<TASK-ID>/requirements.md`.
5. Create or switch to the task branch with the correct prefix.
6. Report branch, workspace path, and the next workflow step.

## Do Not

- start implementation
- hide a dirty worktree
